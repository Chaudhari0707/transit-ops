## Scaffold note (migration intent)

Current code has `admin_users` (username login). Architecture **replaces** that with **Better Auth** core tables (`user`, `session`, `account`, `verification`) + app `additionalFields` (`role`, etc.). Implementation must migrate/remove `admin_users` when coding starts — not dual-run two auth systems.
